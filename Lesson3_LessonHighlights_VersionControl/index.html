Highlight #1 Version Control System(VCS)

A Version control system is a type of management software that records all changes to a file or set of files over time. Thus, allowing one to reference specific versions when needed. Also, VCS' can be very helpful for development teams.

Highlight #2 Why use VCS?

VCS' allow developers to make numerous modifications to a project without getting unorganized and losing files. Great for development teams on account that VCS' show who changed the files, when they were changed, what was changed, and how they were merged. Therefore, helping multiple user projects to stay organized.

Highlight #3 VCS local repository vs. remote repository

A local repository is a repository that is stored on a local machine. Mainly used for individual projects. As for a remote repository, that is a repository which is stored on a remote server and can be accessed by numerous team members.

Highlight #4 VCS Benefits for multiple user projects

Using VCS' for multiple user projetcs can yield many great benefits. Firstly, multiple users can work on the same files at once and without interference. Next, files cannot be overwritten, which is crucial for large projects. Lastly, no matter how long ago a certain version of a file was stored, it can always be recalled.

Highlight #5 Git

Git, is a commonly used version control system (VCS). Git is very similiar to most VCS', however, it differs in the sense that Git acts as a mini filesystem with powerful tools built into it, rather than just a VCS. Furthermore, most VCS' store their information as a list of file based changes, although with git, a picture is taken of the information and compared to photos of past saves.
